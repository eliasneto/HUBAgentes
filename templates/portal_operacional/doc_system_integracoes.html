Integracoes

Guia operacional

Como cadastrar e validar as conexoes do sistema sem expor credenciais ou depender de tentativa e erro.

Esta tela documenta o modulo de Integracoes do portal. Ela explica os tipos de conexao, os campos aceitos em cada cadastro, exemplos de preenchimento e os principais cuidados de seguranca e validacao.

DOC integracoes

Passo a passo

Fluxo recomendado para criar uma conexao

  1. Abra Administrador > Integracoes e clique em Nova conexao.
  2. Escolha o tipo: IA, Google Drive ou Pasta local.
  3. Preencha nome e status da conexao.
  4. Informe os campos tecnicos obrigatorios do tipo escolhido.
  5. Salve o cadastro.
  6. Use o botao Validar conexao para confirmar a comunicacao com o sistema externo ou com a pasta autorizada.
  7. Se a validacao falhar, revise a credencial, o caminho ou as permissoes antes de liberar uso pelos agentes.

Tipos

Quais conexoes existem hoje

IA

Usada pelos agentes para conversar com o provedor de inteligencia artificial.

Exemplos de provedor: OpenAI, Anthropic, Gemini.
Google Drive

Usada para acessar pastas em nuvem compartilhadas com a conta tecnica.

Depende de service account e credenciais JSON validas.
Pasta local

Autoriza o sistema a ler uma pasta existente no servidor.

Hoje o escopo operacional aceito nessa conexao continua focado em arquivos PDF.

Windows

Digite o caminho no formato Windows normalmente: C:\HubAgentes\contratos. O sistema converte automaticamente para o formato interno do servidor.

Mapeamento

A pasta C:\HubAgentes corresponde a /app/entradas dentro do container. Qualquer subpasta criada no Windows fica acessivel com o prefixo /app/entradas/.

Observacoes importantes

Cuidados antes de validar ou liberar para uso

  • O botao Validar conexao deve ser usado sempre depois de criar ou editar uma conexao.
  • Em edicao, campos secretos como Chave de API e Credenciais JSON podem ficar em branco para preservar o valor atual.
  • Para Google Drive, a pasta precisa estar compartilhada com o e-mail da service account.
  • Para pasta local, o caminho informado precisa existir no servidor. Use C:\HubAgentes\... (Windows) ou /app/entradas/... (servidor direto).
  • Com Ler subpastas ativo, o sistema usa rglob para varrer toda a arvore de subpastas — ideal para grandes volumes organizados por pasta.
  • No escopo atual, a conexao de pasta local permanece orientada a PDF.
  • Uma conexao inativa ou com erro pode impedir uso por agentes ou fontes documentais vinculadas.
  • A listagem do portal exibe apenas metadados operacionais; credenciais sensiveis continuam ocultas.

Extensoes suportadas

Formatos de entrada e saida aceitos pelo sistema

Entrada — todos os provedores

PDF — documentos, contratos, editais, relatorios.
TXT — arquivos de texto puro.
CSV — dados tabulares separados por virgula.
PNG / JPG / JPEG — imagens (requer provedor com suporte visual, ex: Gemini).
XLSX — planilhas Excel (requer provedor com suporte a binario).

Configure as extensoes aceitas no cadastro de cada Integracao de pasta local.
Entrada — por provedor

Gemini — PDF, TXT, CSV, PNG, JPG, XLSX
OpenAI — PDF, TXT, CSV, PNG, JPG
Groq — somente TXT e CSV (texto puro, sem binarios)
Anthropic — PDF, TXT, CSV, PNG, JPG

Groq nao suporta arquivos binarios (PDF, imagens, Excel). Use Gemini para esses formatos.
Saida — todos os provedores

JSON — estrutura de dados bruta retornada pela IA.
Excel (.xlsx) — planilha com dados tabulados.
CSV — valores separados por virgula.
PDF — documento formatado gerado pelo sistema.
TXT — texto simples.
Definido pela IA — a IA informa o formato via campo formato_saida.

O formato de saida e configurado no agente e gerado pelo sistema — independe do provedor de IA.
Resumo por provedor 
Gemini   → entrada: PDF, TXT, CSV, PNG, JPG, XLSX | saida: qualquer
OpenAI   → entrada: PDF, TXT, CSV, PNG, JPG       | saida: qualquer
Anthropic→ entrada: PDF, TXT, CSV, PNG, JPG       | saida: qualquer
Groq     → entrada: TXT, CSV apenas               | saida: qualquer
