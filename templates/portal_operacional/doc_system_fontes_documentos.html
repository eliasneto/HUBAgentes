Fontes de documentos

Guia operacional

Como cadastrar, revisar e manter as origens documentais que alimentam a leitura dos agentes.

Esta tela documenta o modulo de Fontes de documentos do portal. Ela explica os tipos de origem, os campos do cadastro, exemplos de preenchimento e os cuidados para garantir que os documentos certos cheguem na esteira operacional.

DOC fontes

Passo a passo

Fluxo recomendado para cadastrar uma fonte

  1. Abra Administrador > Fontes de documentos.
  2. Clique em Nova fonte ou escolha diretamente Nova fonte Google Drive ou Nova fonte local.
  3. Defina o tipo da fonte documental.
  4. Preencha nome, status e os campos especificos da origem.
  5. Salve a fonte.
  6. Revise a listagem para confirmar se a origem ficou visivel com o tipo correto.

Tipos

Quais origens documentais existem hoje

Google Drive

Usada quando os documentos ficam em uma pasta na nuvem compartilhada com a conta tecnica.

Ideal para fluxos colaborativos e recebimento centralizado.
Pasta local

Usada quando os documentos estao em uma pasta existente no servidor ou no ambiente autorizado.

Ideal para integracoes locais, pastas montadas ou volumes internos.

Pasta local — como funciona o mapeamento

De onde vem a pasta /app/entradas

O sistema roda dentro de um container Docker. Para que os documentos do Windows fiquem acessiveis, a pasta C:\HubAgentes do computador e mapeada automaticamente para /app/entradas dentro do container.

Windows

Crie e organize suas pastas em C:\HubAgentes\. Coloque os PDFs nelas normalmente pelo Windows Explorer.

Exemplo: C:\HubAgentes\contratos\
Sistema

No cadastro de pasta local, use o caminho correspondente no container, substituindo C:\HubAgentes por /app/entradas.

Exemplo: /app/entradas/contratos
Tabela de correspondencia de caminhos 
C:\HubAgentes\                      →  /app/entradas/
C:\HubAgentes\contratos\            →  /app/entradas/contratos
C:\HubAgentes\relatorios\           →  /app/entradas/relatorios
C:\HubAgentes\clientes\empresa-xyz\ →  /app/entradas/clientes/empresa-xyz

O que aparece na listagem

Como interpretar os cards da tela principal

Google Drive

Mostra nome, status, pasta vinculada, conexao usada, quantidade de itens sincronizados e ultima validacao.

Pasta local

Mostra nome, status, caminho base, extensoes permitidas, leitura recursiva e ultima validacao.

Editar

Permite ajustar a fonte ja cadastrada sem precisar recriar a origem inteira.

Estado vazio

Quando nao ha fontes cadastradas, a tela mostra o estado vazio para orientar o primeiro cadastro.

Observacoes importantes

Cuidados antes de liberar uma fonte para uso

  • Uma fonte documental depende de uma conexao valida e ativa quando o tipo usa Google Drive.
  • Para Google Drive, a pasta deve estar compartilhada com a conta tecnica correta.
  • Para pasta local, o caminho informado deve sempre comecar com /app/entradas/ — nunca use caminhos Windows como C:\..., pois o sistema roda em Linux (container Docker).
  • A pasta C:\HubAgentes no Windows corresponde a /app/entradas no container. Crie suas subpastas dentro de C:\HubAgentes e referencie-as com o prefixo /app/entradas/ no cadastro.
  • Ativar leitura recursiva amplia o alcance da busca para subpastas, entao use com intencao.
  • Revise o tipo da fonte com cuidado, porque isso muda a forma como o sistema localiza os documentos.
  • As fontes de documentos sao a base da entrada operacional; se a origem estiver errada, o agente pode processar arquivos incorretos ou nao encontrar nenhum documento.
