Otimizacao de Custos de IA

Eficiencia operacional

Como o sistema reduz o consumo de tokens e o que voce pode fazer para economizar ainda mais creditos de IA.

Cada chamada para um provedor de IA consome tokens — a unidade de cobranca da maioria das APIs. Tokens entram (prompt + documentos) e saem (resposta gerada). Esta pagina descreve as otimizacoes implementadas no HUB Agentes e como elas funcionam em cada provedor.

$ tokens

Conceito basico

O que sao tokens e por que importam

Um token equivale aproximadamente a 3–4 caracteres de texto. Cada execucao de agente consome tokens de entrada (o que voce envia: prompt + documento) e tokens de saida (o que a IA responde). Os provedores cobram por milhao de tokens — quanto mais tokens, maior o custo.

Tokens de entrada

Incluem o prompt do agente, instrucoes de formato e o conteudo dos documentos enviados. Sao a maior fonte de custo em lotes grandes.

Tokens de saida

O texto gerado pela IA como resposta. Geralmente mais caro por token do que a entrada, mas representa uma parcela menor do total.

Tokens em cache

Alguns provedores (Anthropic) oferecem desconto de ate 90% em tokens que nao mudaram desde a ultima chamada — o prompt cacheado.

Como acompanhar

Cada card de processamento exibe o total de tokens consumidos. Acesse Configuracao de Custos para ver o historico detalhado por execucao.

Otimizacao 1 — Maior impacto

Prompt Caching (cache de instrucoes)

Quando o mesmo agente processa varios documentos, o prompt base nao muda entre as chamadas — so o documento e diferente. O prompt caching permite que o provedor reutilize o processamento ja feito do prompt, cobrando muito menos pelos tokens repetidos.

Como o sistema usa o cache (Anthropic) 
Chamada 1 — processa documento A:
  system: [prompt do agente] → MISS (cobra 100%)
  user:   [conteudo do documento A]

Chamada 2 — processa documento B:
  system: [prompt do agente] → HIT (cobra ~10%)
  user:   [conteudo do documento B]

Chamada N — mesmo agente, novo documento:
  system: [prompt do agente] → HIT (cobra ~10%)
  user:   [conteudo do documento N]
Para maximizar o cache 
- Execute lotes grandes em sequencia rapida
  (o cache expira em 5 min sem uso — Anthropic)
- Mantenha o prompt base estavél entre execucoes
- Evite alterar o prompt entre documentos do lote
- Use modelos maiores: prompts mais longos
  geram maior economia proporcional em cache

Otimizacao 2

Remocao de nome de arquivo duplicado

Versoes anteriores do sistema incluiam o nome do arquivo duas vezes em cada chamada — uma vez dentro do bloco do documento e novamente como sufixo no prompt (Arquivo analisado: nome.pdf). Em lotes com dezenas de arquivos, essa redundancia acumulava centenas de tokens desperdicados.

Antes (redundante) 
[Arquivo: relatorio.pdf]
<conteudo do documento>

[prompt do agente]

Arquivo analisado: relatorio.pdf  ← repetido
Depois (otimizado) 
[Arquivo: relatorio.pdf]
<conteudo do documento>

[prompt do agente]  ← nome aparece so uma vez
  • Provedores afetados: Anthropic, OpenAI, Gemini. O Groq ja nao tinha essa duplicacao.
  • Economia estimada: 50–150 tokens por documento, dependendo do tamanho do nome do arquivo.
  • Em um lote de 100 documentos: ~5.000–15.000 tokens economizados por execucao.

Otimizacao 3

Instrucao de formato AI_DEFINED enxuta

Quando um agente usa o formato de saida Definido pela IA, o sistema adiciona automaticamente instrucoes explicando os formatos disponiveis e o schema JSON esperado. Esse bloco foi reduzido de ~500 tokens para ~100 tokens sem perda de clareza.

Antes (~500 tokens) 
Antes de responder, escolha o formato de saida mais
adequado ao conteudo que voce vai gerar, seguindo
os criterios abaixo:

| Formato | Use quando a resposta for...         |
|---------|--------------------------------------|
| xlsx    | Dados tabulares: listas, comparativos|
| csv     | Dados tabulares simples...           |
| pdf     | Relatorio narrativo, analise...      |
| json    | Dados estruturados para APIs...      |
| txt     | Texto corrido simples...             |

Responda OBRIGATORIAMENTE com um JSON valido...
{
  "formato_saida": "...",
  "justificativa": "...",
  "dados": ...
}
Regras:
- "dados" deve conter o conteudo completo...
[+ 4 regras adicionais]
Depois (~100 tokens) 
Escolha o formato mais adequado: xlsx (tabelas),
csv (dados simples), pdf (HTML com CSS),
json (integracao), txt (texto puro).

Responda EXCLUSIVAMENTE com JSON valido:
{"formato_saida": "...", "justificativa": "...",
 "dados": <conteudo completo>}

Regra: dados completo, sem truncar. xlsx/csv:
lista de listas com cabecalhos na 1a linha.
  • Economia: ~400 tokens por execucao que usa formato AI_DEFINED.
  • Aplicado a todos os 4 provedores (a instrucao vai no prompt, nao na API).
  • O comportamento da IA permanece identico — os formatos e o schema JSON nao mudaram.

Exemplo combinado: agente Anthropic processando 100 documentos com formato AI_DEFINED. Prompt de 2.000 tokens + nome do arquivo (80 tokens) + bloco AI_DEFINED (500 tokens) = 2.580 tokens de entrada por documento. Boas praticas

Como voce pode economizar ainda mais

Prompts concisos

Cada palavra no prompt e um token. Instrucoes claras e diretas economizam mais do que prompts longos com exemplos redundantes.

Lotes grandes, nao pequenos

Processar 100 documentos de uma vez aproveita o cache do prompt por mais tempo do que 10 lotes de 10 — o prompt so e cobrado integralmente na primeira chamada.

Escolha o modelo certo

Modelos menores (ex.: Haiku, Flash) custam 10–20x menos que os maiores. Para tarefas simples de extracao ou classificacao, eles produzem resultados equivalentes.

Limite de tentativas

Configure o campo Maximo de tentativas por documento no agente. Sem limite, um documento com erro permanente seria re-executado indefinidamente, consumindo tokens a cada tentativa.

Formato fixo vs AI_DEFINED

Se voce ja sabe o formato de saida esperado, configure-o diretamente no agente (JSON, Excel, etc.). O formato AI_DEFINED adiciona tokens de instrucao extras a cada chamada.

Documentos grandes

Documentos acima do contexto do modelo falham e cobram pelo processamento parcial. Divida documentos muito grandes antes de enviar, ou use um modelo com contexto mais amplo.

Recomendacao padrao

Use gemini-3.1-flash-lite no campo Modelo padrao da integracao Gemini. Com 500 requisicoes/dia gratis, cobre o uso tipico do HUB Agentes sem custo.

Embedding vs Gerativo

Modelos como Gemini Embedding 1 e Gemini Embedding 2 convertem texto em vetores numericos — eles nao geram respostas. Nao devem ser usados no campo Modelo padrao de uma integracao de IA.

RPD vs RPM

RPD = requisicoes por dia (limite diario). RPM = requisicoes por minuto (limite de velocidade). Para lotes grandes, o RPM pode ser o gargalo antes do RPD.

Geracao importa

O gemini-3.1-flash-lite supera o gemini-2.0-flash em qualidade apesar do nome "lite" — pertence a uma geracao mais nova com arquitetura melhorada.
