Busca API Google Drive

Guia passo a passo

Como criar uma Service Account no Google Cloud e conectar ao sistema.

O sistema acessa pastas do Google Drive usando uma conta tecnica (Service Account) — sem senha interativa. Voce compartilha a pasta com o e-mail da conta tecnica e o sistema passa a ler os arquivos dela automaticamente.

Visao geral

Como o sistema acessa o Google Drive

Google Cloud Console
  └── Projeto
        └── API Google Drive (ativada)
              └── Service Account
                    ├── E-mail  (ex: hub-agentes@meu-projeto.iam.gserviceaccount.com)
                    └── Chave JSON (credencial privada)
                          │
                          ▼
                    HUB Agentes
                    └── Integracao Google Drive
                          └── Fontes de Documento (pastas compartilhadas)

Passo a passo

Fluxo completo de configuracao

  1. Criar (ou selecionar) um projeto no Google Cloud Console.
  2. Ativar a Google Drive API no projeto.
  3. Criar uma Service Account e copiar o e-mail gerado.
  4. Gerar e baixar a chave JSON da service account.
  5. Compartilhar cada pasta do Drive com o e-mail da service account.
  6. Criar a integracao no sistema preenchendo e-mail e JSON.
  7. Adicionar as pastas como fontes de documento.

Passo 1

Criar um projeto no Google Cloud

  • Acesse console.cloud.google.com.
  • No topo, clique no seletor de projeto e depois em Novo projeto.
  • Dê um nome ao projeto (ex: hub-agentes) e clique em Criar.
  • Aguarde e selecione o novo projeto no seletor.
Dica

Se sua organizacao ja tem um projeto Google Cloud, voce pode reutiliza-lo. Basta seleciona-lo no seletor antes de continuar.

Passo 2

Ativar a API do Google Drive

  • Com o projeto selecionado, acesse APIs e Servicos > Biblioteca.
  • Pesquise por Google Drive API.
  • Clique no resultado e depois em Ativar.
  • Aguarde a ativacao — voce sera redirecionado para a pagina da API.

Passo 3

Criar a Service Account

  • Acesse APIs e Servicos > Credenciais.
  • Clique em + Criar credenciais e selecione Conta de servico.
  • Preencha o nome (ex: hub-agentes) e clique em Criar e continuar.
  • Nas etapas seguintes de permissoes, clique em Continuar e depois em Concluido.
  • A conta aparecera listada em Credenciais > Contas de servico.
Importante

Copie o e-mail da service account (ex: hub-agentes@meu-projeto.iam.gserviceaccount.com). Voce vai precisar dele para compartilhar as pastas e preencher o cadastro no sistema.

Passo 4

Gerar a chave JSON

  • Na lista de contas de servico, clique no e-mail da conta criada.
  • Acesse a aba Chaves.
  • Clique em Adicionar chave > Criar nova chave.
  • Selecione o tipo JSON e clique em Criar.
  • O arquivo .json sera baixado automaticamente para o seu computador.
Estrutura do arquivo JSON (resumida) 
{
  "type": "service_account",
  "project_id": "meu-projeto",
  "private_key_id": "abc123...",
  "private_key": "-----BEGIN RSA PRIVATE KEY-----\n...",
  "client_email": "hub-agentes@meu-projeto.iam.gserviceaccount.com",
  "client_id": "123456789",
  "auth_uri": "https://accounts.google.com/o/oauth2/auth",
  "token_uri": "https://oauth2.googleapis.com/token",
  ...
}
Seguranca

Este arquivo contem a credencial privada. Guarde-o em local seguro e nunca publique em repositorios Git ou compartilhe por e-mail. O sistema armazena o conteudo criptografado.

Passo 5

Compartilhar a pasta do Drive com a Service Account

  • Abra o Google Drive com sua conta pessoal ou corporativa.
  • Clique com o botao direito na pasta que os agentes vao usar e selecione Compartilhar.
  • No campo de pessoas, cole o e-mail da service account.
  • Defina a permissao como Leitor (somente leitura e suficiente).
  • Desmarque Notificar pessoas — a service account nao recebe e-mails.
  • Clique em Compartilhar.
Multiplas pastas

Repita este passo para cada pasta que quiser adicionar como fonte de documento no sistema. Uma unica service account pode ser compartilhada em quantas pastas forem necessarias.

Acesso restrito

A service account so acessa pastas explicitamente compartilhadas com o e-mail dela. O restante do Drive permanece inacessivel.

Passo 6

Criar a integracao no sistema

  • Acesse Administrador > Integracoes e clique em Nova conexao.
  • Selecione o tipo Google Drive.
  • Preencha os campos conforme a tabela abaixo.
  • Salve e use o botao Validar conexao para confirmar a comunicacao.
Campo
O que preencher
Nome
Nome descritivo (ex: Google Drive - Licitacoes)
Status
Ativa
E-mail da Service Account
O e-mail copiado no Passo 3 (ex: hub-agentes@projeto.iam.gserviceaccount.com)
Credencial JSON
Cole o conteudo completo do arquivo .json baixado no Passo 4 — incluindo as chaves { e }
Extensoes permitidas
Deixe em branco para usar o padrao (pdf), ou informe as extensoes desejadas

Passo 7

Adicionar fontes de documento

  • Acesse Operacao > Fontes de Documentos e clique em Nova fonte.
  • Selecione a integracao Google Drive criada no Passo 6.
  • Cole a URL da pasta do Google Drive.
  • O sistema extrai o ID da pasta automaticamente a partir da URL.
  • Dê um nome para a fonte e salve.
Formato da URL da pasta 
https://drive.google.com/drive/folders/1A2B3C4D5E6F7G8H9I0J
                                              ↑ este trecho e o ID da pasta (extraido automaticamente)
Proxima etapa

Apos criar a fonte, associe-a a um agente na tela Gerenciar Agentes, campo Origem padrao.

Perguntas frequentes

Duvidas comuns

  • Preciso criar uma service account por pasta? Nao. Uma unica service account pode ser compartilhada em quantas pastas quiser. Voce compartilha cada pasta com o mesmo e-mail e cadastra cada uma como uma fonte de documento separada.
  • A service account tem acesso a todo o meu Drive? Nao. Ela acessa apenas as pastas e arquivos explicitamente compartilhados com o e-mail dela. O restante do Drive permanece inacessivel.
  • Preciso renovar a chave JSON? Nao automaticamente. A chave JSON nao expira, mas voce pode revoga-la a qualquer momento no Google Cloud Console e gerar uma nova se necessario.
  • O sistema modifica ou exclui arquivos do Drive? Nao. O sistema usa permissao de Leitor — ele le os arquivos para processamento mas nunca altera, move ou exclui nada no Drive.
